Waze Churn Prevention Project: Exploratory data analysis, Hypothesis testing and Regression analysis

Background on the Waze Scenario:

Waze's free navigation app simplifies travel for drivers worldwide. The collaborative effort of map editors, beta testers, translators, partners, and users ensures each journey is optimized for safety and efficiency. Collaborations with cities, transportation authorities, broadcasters, businesses, and first responders further Waze's commitment to facilitating safer and more efficient travel experiences.

The project employs a dataset titled 'waze_dataset.csv,' meticulously curated through a partnership between Google and Waze, containing synthetic data exclusively generated for this collaborative initiative.

Exploratory data analysis

I performed in-depth exploratory data analysis for the churn project, employing various tools to create insightful visuals. These visualizations not only helped us gain a deeper understanding of the data but also revealed key insights into user behavior, aiding our decision-making process

Hypothesis testing

I explored hypotheses within the churn dataset, examining key differences. Specifically, I devised a two-sample hypothesis test to scrutinize the mean number of rides across different device types - Android versus iPhone. Delving into statistical significance, I aimed to uncover compelling insights into user behavior and device preferences.

  • Explore Project Details

Regression analysis

I used user data to create and evaluate a binomial logistic regression model. This involved crafting relevant features, checking for multicollinearity, building the model, and assessing its performance. The model's results help reveal important relationships between variables and predict binary outcomes based on the data
